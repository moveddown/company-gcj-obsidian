

# **ARM Cortex-M0+ 内核自带的寄存器模块**

![[Pasted image 20260602161223.png|300]]

| 项目            | 作用                 |
| ------------- | ------------------ |
| NVIC          | 管中断开关、挂起、优先级       |
| SCB           | 管 CPU 系统状态、复位、异常配置 |
| SysTick       | 内核自带定时器，常做 1ms 心跳  |
| Fault Reports | 看 CPU 异常/故障状态      |

### NVIC

![[Pasted image 20260602162157.png]]

| 列           | 意思                  |
| ----------- | ------------------- |
| Idx         | 中断号                 |
| Source      | 中断来源说明              |
| Name        | 中断名，对应代码里的 xxx_IRQn |
| E           | Enable，是否使能这个中断     |
| P           | Pending，是否有挂起的中断请求  |
| Priority    | 中断优先级，数值越小优先级越高     |
| VECTACTIVE  | 当前正在执行的异常/中断编号      |
| VECTPENDING | 当前等待处理的异常/中断编号      |
如果外设产生中断，但 CPU 还没处理，P 会变成 1。

==这个窗口主要是看，中断有无打开，有没有挂起，CPU当前是不是正在处理中断==


### SCB

![[Pasted image 20260602162831.png]]
系统控制块
SCB->CCR  配置控制寄存器
SCB->SCR  系统控制寄存器

| 列           | 意思                                                                            |
| :---------- | ----------------------------------------------------------------------------- |
| UNALIGN_TRP | 非对齐访问陷阱。<br><br>如果勾上，CPU 遇到某些“地址没对齐”的访问会触发异常。                                 |
| STKALIGN    | 异常入栈 8 字节对齐。<br><br>中断/异常发生时，CPU 会自动把一些寄存器压入栈。STKALIGN=1 表示异常入栈时保持 8 字节对齐。    |
| SLEEPONEXIT | 中断退出后是否立刻回到睡眠。<br><br>- 0：中断退出后回主程序<br>- 1：中断退出后直接再次进入睡眠                      |
| SLEEPDEEP   | 深度睡眠选择。<br><br>- 0：进入普通 Sleep<br>- 1：进入 Deep Sleep / Stop / Standby 这类更深低功耗模式 |
| SEVONPEND   | 有中断挂起时产生事件。<br><br>这个和 WFE 等待事件机制有关，普通中断调试基本不用管。                              |
==**SCB 不是具体外设，它是 CPU 内核级配置；你现在最需要关心的是 SLEEPDEEP，因为它会决定 WFI 后是普通睡眠还是深度睡眠。**==

### SysTick

![[Pasted image 20260602164034.png]]
系统滴答定时器，最常用是每隔固定时间进一次中断。所以[[01 统一心跳-模版功能代码|统一心跳]]会用到


**Control & Status**

| 列         | 意思                                                                 |
| :-------- | ------------------------------------------------------------------ |
| ENABLE    | SysTick 是否启动- 1：计数器正在工作<br>- 0：停止                                  |
| CLKSOURCE | SysTick 时钟来源,- 1：使用 CPU 内核时钟<br>- 0：使用外部/参考时钟，具体看芯片实现              |
| TICKINT   | 计数到 0 时是否触发 SysTick 中断。- 1：计数到 0 会进入 SysTick_Handler()- 0：只计数，不进中断 |
| COUNTFLAG | 计数到 0 标志。<br><br>- 1：从上次读取后，计数器曾经倒数到 0<br>- 读 CTRL 后通常会清掉          |
**Reload & Current Value**


| 列    | 意思                                              |
| ---- | ----------------------------------------------- |
| LOAD | 重装载值。<br><br>SysTick 是倒计数器，从 LOAD 往下数到 0，然后重新装载 |
| VAL  | 当前计数值                                           |
|      |                                                 |

==**这个窗口用来看 SysTick 是否在跑、周期是多少、会不会进 SysTick_Handler。你现在它在跑，但不会进中断。**==


### Fault Reports

![[Pasted image 20260602165154.png]]
**CPU 调试状态/异常状态查看窗口**


|标志|意思|
|---|---|
|HALTED|CPU 因为调试 halt 停住|
|BKPT|执行了断点指令 BKPT|
|DWTTRAP|DWT 调试单元触发，M0+ 上一般很少用|
|VCATCH|向量捕获触发，比如异常入口被调试器捕获|
|EXTERNAL|外部调试请求让 CPU 停住|
==**这个窗口是查“CPU 为什么被调试器停住/异常捕获”的，平时调 GPIO、I2C、SysTick、中断时基本不用管**==
