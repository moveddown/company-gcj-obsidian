## AI 辅助画 PCB 工具选择

### 结论

目前还没有成熟到可以完全替代人工的“AI 自动画板”工具。更稳的方式是：AI 负责[[选型]]、[[原理图审查]]、低功耗风险检查、PCB 布局布线建议；关键电路和最终打板文件仍然由人工确认。

如果目标是快速在[[嘉立创]]下单、贴片，优先用[[嘉立创EDA]]；如果目标是接入 AI、脚本自动化、版本管理和长期工程积累，[[KiCad]] 更适合。

### 嘉立创 EDA

适合生产落地，尤其是配合立创商城器件库、封装库、BOM、贴片下单流程。

可用能力：

- 嘉立创 EDA 自带自动布线，适合简单低速板，但复杂板不建议完全依赖。
- 扩展广场有“PCB 自动化工具”，可用于自动扇出、短线自动布线、模块化布局、DFM 审核等。
- Run API Gateway 可以让 OpenCode、QwenCode、KimiCode 等 AI 编程工具通过 WebSocket 调用嘉立创 EDA 扩展 API，但配置门槛较高。

不足：

- 目前没有真正成熟的一句话 AI 自动生成完整原理图和 PCB 的工作流。
- AI 接入能力不如 KiCad 开放。
- 自动布线结果通常还需要人工调整。

建议工作流：

1. AI 帮忙做器件选型和模块方案。
2. 在嘉立创 EDA 中手动画原理图。
3. AI 帮忙审查原理图、电源、低功耗倒灌、接口、上拉下拉、下载口等问题。
4. PCB 关键部分手动布局布线。
5. 普通低速线可尝试自动布线或 PCB 自动化插件。
6. 最后做 DRC、DFM、BOM 和封装核对。

### KiCad

适合 AI 辅助和工程自动化。KiCad 的 `.kicad_sch`、`.kicad_pcb` 文件格式开放，适合脚本、Git 和 AI agent 读取修改。

可关注工具：

- [[Quilter]]：支持 KiCad 工程，偏 AI 自动布局和布线，可生成候选 PCB layout。
- [[ProtoFlow]]：偏早期方案生成，可导出到 KiCad / Altium，适合生成草稿。
- [[Circuit Weaver]]：spec-first 电路生成工具，可生成 KiCad 相关设计文件，属于较新的方向。
- KiCad MCP / AI Copilot 类工具：让 AI 通过 MCP 操作 KiCad，适合会折腾自动化环境的人。
- [[FreeRouting]]：传统自动布线器，不是 AI，但可配合 KiCad 处理普通低速信号线。

优点：

- 文件格式开放，AI 和脚本更容易接入。
- 适合 Git 版本管理。
- 社区工具多。
- 适合长期维护自己的工程模板和封装库。

缺点：

- 国内器件库、立创商城、贴片下单流程没有嘉立创 EDA 顺手。
- 新手更容易在封装、BOM、坐标文件上踩坑。
- AI 生成的器件、封装和网络连接必须人工复核。

建议工作流：

1. KiCad 画原理图。
2. AI 审查原理图，包括电源、低功耗倒灌、I2C/SPI/UART、复位、BOOT、下载口、4G 供电峰值等。
3. 人工手动摆放关键器件，例如 MCU、电源、晶振、天线、采样电路、接口。
4. 尝试 Quilter 或 FreeRouting 辅助普通信号线。
5. 人工检查 DRC、线宽、间距、回流路径、封装方向、丝印和安装孔。
6. 导出 Gerber、钻孔、BOM、坐标文件后去嘉立创下单。

### 实际选择

- 追求最快打样和贴片：选[[嘉立创EDA]]。
- 追求 AI 辅助、自动化、可版本管理：选[[KiCad]]。
- 新手做小批量嵌入式板：嘉立创 EDA 更省事。
- 想长期积累自己的工具链：KiCad 更值得投入。

一句话：现在最靠谱的不是“AI 全自动画板”，而是“AI 辅助选型和审图 + 人工把关关键电路 + EDA 工具辅助布线”。
