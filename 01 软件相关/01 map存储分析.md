
# 一、解释关键字
![[Pasted image 20260528131459.png]]

Code : 代码程序大小，函数，关键字，中断服务函数，驱动代码，库函数，机器指令 。 占FLASH

RO-Date：read only data 只读数据，const 全局变量，字符串常量，只读数组，常量表格。占FLASH

RW-Date: Read Write Data 已经初始化读写变脸，就是全局变量，还有static变量赋值过的。 初始值是在FLASH，运行时候会复制到RAM

ZI-Date:  Zero Initialized Data 零初始化变量，未初始化的全局/静态变量。初始化为0的全局静态变量。栈和堆也在这个里面。 只占RAM





# 二、实际使用情况
结合kile5来分析
![[Pasted image 20260529091352.png]]
上图配置的flash，是0xF400 , 换算一下就是61k的空间 （62464/1024）

然后为什么是这个呢，首先是起始地址
![[Pasted image 20260529091946.png]]
这个可以看到flash和RAM的地址，然后大小可以就可以设置为64k的大小，也就是（64\*1024=0x10000） 。上图之所以设置的是FC00。是因为后面的几个kb是留着写入一些固定的FLASH参数。因为这款芯片是512字节为一页，所以一共128页（0x10000 / 512 ），使用了后面的122几页，所以最大的FLASH就是122页，也就是0xF400(122*512)

当清楚了自己的配置后，我们在看一下这个map里面
![[Pasted image 20260529095030.png]]
这个是最后的总结，可以看到FLASH 已经有60.10kb。结合我们当时设置的0xFC00，也就是63kb的空间，可以知道还有2.9kb的FLASH空间。然后比如RAM可以看到是7.16kb。我们设置的是0x2000
