“外设通信脚防倒灌”，重点是防止这种情况：

**外设已经断电了，但 MCU 的通信引脚还在输出高电平，电流从 MCU 引脚流进外设内部，最后流到外设的 VCC 上。**

这就叫倒灌、反向供电、back-power。

典型场景：

```text
MCU 还供电 3.3V
外设 VCC 已关闭 0V

MCU_TX 输出高电平 3.3V
        |
        v
外设_RX 引脚
        |
   外设内部保护二极管/ESD结构
        |
        v
外设 VCC
```

结果是：你以为外设断电了，但它可能被 MCU 的 IO 偷偷“喂电”。轻则低功耗电流变大，重则外设半死不活、启动异常，甚至损伤器件。

**哪些脚容易倒灌**

常见是 MCU 输出到外设的脚：

```text
UART_TX
SPI_SCK
SPI_MOSI
SPI_CS
I2C_SCL/SDA
模块_EN
模块_RESET
传感器_INT，有时也会反过来倒灌 MCU
```

尤其是这些情况要小心：

```text
MCU 供电没关，外设电源被 MOS 管关掉
MCU 进入低功耗，外设完全断电
外设和 MCU 不是同一个电源域
外设 VCC 比 MCU VCC 更早掉电
```

**为什么高电平危险**

很多芯片 IO 口内部都有 ESD 保护结构，可以简化理解成这样：

```text
IO 引脚 ----|<|---- VCC
IO 引脚 ----|>|---- GND
```

正常工作时它们不导通。

但如果外设 VCC 是 0V，而外设 RX 引脚被 MCU 拉到 3.3V，IO 电压比外设 VCC 高太多，内部保护结构就可能导通：

```text
MCU_TX = 3.3V
外设_VCC = 0V

电流路径：
MCU_TX -> 外设_RX -> 外设内部保护二极管 -> 外设_VCC -> 外设内部电路/GND
```

这个电流可能只有几十微安，也可能几毫安，取决于外设内部结构和外部电路。

**怎么处理**

最常用的处理方式有几类。

**1. 外设断电前，把 MCU 输出脚拉低**

比如 UART：

```text
MCU_TX -> 外设_RX
```

外设断电前，先让 `MCU_TX` 输出低，或者改输入高阻。

```c
// 关闭外设电源前
UART_DeInit();
GPIO_OutputLow(MCU_TX_PIN);
Power_Off_Sensor();
```

低电平通常不会通过外设 IO 倒灌到 VCC。

**2. 把通信脚改成输入/模拟输入高阻**

如果这个脚不需要保持电平，可以在低功耗前释放掉：

```c
UART_DeInit();
GPIO_ConfigAnalog(MCU_TX_PIN);
GPIO_ConfigAnalog(MCU_RX_PIN);
Power_Off_Module();
```

这样 MCU 不再主动输出高电平。

但注意：高阻不是永远最好。如果外设某个脚必须保持确定状态，例如 `CS`、`EN`、`RESET`，那就不能随便高阻。

**3. 控制脚保持“关闭态”**

比如模块电源使能：

```text
EN = 1 外设上电
EN = 0 外设断电
```

低功耗时应该明确输出低：

```c
GPIO_OutputLow(MODULE_EN_PIN);
```

不要改成模拟输入。因为高阻后 EN 可能被干扰拉高，模块又偷偷上电。

再比如复位脚：

```text
RESET = 0 外设复位
RESET = 1 外设运行
```

低功耗时如果希望外设保持关闭/复位，通常让它明确保持低电平。

**4. I2C 要特别看外部上拉**

I2C 的 `SCL/SDA` 通常外部有上拉电阻：

```text
3.3V -> 4.7k -> SDA
3.3V -> 4.7k -> SCL
```

如果 I2C 上拉电源和外设电源是同一个，被关掉了，一般还好。

但如果 I2C 上拉还接在 MCU 的 3.3V，而传感器断电了，就可能这样：

```text
3.3V -> 上拉电阻 -> SDA -> 断电传感器IO -> 传感器VCC
```

这也会倒灌。

所以 I2C 低功耗要确认：

```text
上拉电阻接的是哪个电源？
外设断电后 SDA/SCL 会不会仍被拉高？
传感器手册是否允许 IO 高于 VCC？
```

如果上拉还在 3.3V，而传感器断电，最好加隔离，或者让传感器不断电只休眠，或者把上拉电源也一起关掉。

**实际判断口诀**

可以按这个顺序看每个通信脚：

```text
1. 低功耗时，外设是否还供电？
2. 如果外设断电，这根线会不会被 MCU 或上拉电阻拉高？
3. 如果会被拉高，外设 IO 是否允许在 VCC=0 时承受高电平？
4. 如果不允许，就要拉低、高阻、断上拉，或加隔离。
```

最实用的结论是：

**外设断电时，连接外设的 MCU 输出脚不要保持高电平；有上拉的通信线也要确认上拉电源不会给断电外设倒灌。**

你可以把“防倒灌”理解成一句工程规则：

**外设没电的时候，不要让任何一根接到它的信号线还带着高电平。**
